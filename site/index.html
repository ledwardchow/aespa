Codename AESPA

An LLM-driven automated penetration testing agent.

AESPA (AI-Enabled Security Pentesting Agent) is a FastAPI backend and vanilla-JS UI that drives multi-agent LLM scans against a target. It covers three surfaces — web application, API, and SAST-lite — and runs on your own machine with an LLM provider of your choice.

What it scans

🌐

Web application

Multi-phase crawl — unauthenticated first, then one phase per credential — using parallel Playwright browser workers and LLM-guided BFS. Followed by an agentic dynamic scan in which the Test Lead chooses each action from a toolset.

🔌

API

Parses OpenAPI, Postman, or source specs — or unstructured docs — into an understanding of the API. Runs the same agentic loop without a browser and tracks an OWASP API Top-10 coverage matrix as it goes.

🔍

SAST-lite

Agentic static analysis over an uploaded source ZIP, with file tools path-jailed to the extraction root. Emits leads that seed the dynamic scan rather than standalone reports.

Multi-agent model

Four agent roles per scan

A Test Lead runs the engagement as one continuous session and spawns Specialist agents on high-confidence leads. An Adversarial Validator reviews each finding; a Reporting agent does a post-scan pre-screen. All agents emit status to the Agents panel in the UI.

Test Lead

One continuous session. Builds a recon summary, seeds hypotheses and tasks, and decides each action from its toolset.

Specialist

Spawned on high-confidence leads with a narrow mission. Runs as a parallel asyncio task; cannot recursively dispatch.

Adversarial Validator

Reviews each finding with a disprove-it mandate to reduce false positives. Cannot create findings of its own.

Reporting

Runs a post-scan pre-screen of findings before they reach the report.

A.L.I.C.E. is a separate interactive chat agent for user-directed pentesting.

The central unit of work is a run

Run lifecycle

A web TestRun progresses through created → crawling → crawled → scanning → scanned. Each run owns all its artifacts: pages, traffic, findings, hypotheses, and logs.

  1. 01

    Crawl

    Parallel Playwright workers crawl the target and produce crawled pages plus target-intelligence items.

  2. 02

    Hypothesize

    The Test Lead builds a recon summary and seeds hypotheses and tasks — a work program rather than a fixed checklist.

  3. 03

    Scan

    The agentic loop dispatches Specialist agents to probe and confirm issues over real traffic.

  4. 04

    Validate & report

    Each finding is reviewed by the Adversarial Validator, then the Reporting agent pre-screens before the report.

Measured results

Benchmarks and cost

AESPA has been run against a deliberately-vulnerable target (Bank of Ed) alongside Claude Code, Codex, and local models, and in single-agent vs. multi-agent configurations. As of late May 2026, a full multi-agent scan cost about $7.50 on Sonnet 4.6 token prices and about $1.50 on cheap-model prices.

Running AESPA

Requires Python 3.12+, uv, and an LLM provider key.

uv sync && uv run playwright install chromium && uv run aespa

The UI runs at http://127.0.0.1:8000 by default.

Codename AESPA

AESPA is a research exploration into whether a fully LLM-driven, automated web/API penetration test can work. It is intended to run on a machine you sit in front of: it ships with no authentication, the API is unauthenticated, and it is localhost-first by design. Only point it at targets you are authorized to test.
